## Next.js를 선택한 이유

개발자로 일하던 시절부터 지금까지, 웹 기술은 엄청나게 발전했습니다. 그 중에서도 Next.js는 React 생태계에서 가장 주목받는 프레임워크입니다.

## App Router의 장점

### 1. 서버 컴포넌트

React Server Components는 서버에서 렌더링되어 클라이언트로 전송됩니다.

```tsx
// app/blog/page.tsx - 서버 컴포넌트
async function BlogPage() {
  // 서버에서 직접 데이터 페칭
  const posts = await fetchPosts();
  
  return (
    <div>
      {posts.map(post => (
        <PostCard key={post.id} post={post} />
      ))}
    </div>
  );
}
```

### 2. 중첩 레이아웃

레이아웃을 중첩하여 구성할 수 있습니다.

```
app/
├── layout.tsx          # 루트 레이아웃
├── blog/
│   ├── layout.tsx      # 블로그 레이아웃
│   ├── page.tsx        # /blog
│   └── [slug]/
│       └── page.tsx    # /blog/:slug
```

### 3. 스트리밍과 Suspense

데이터 로딩 중에도 UI를 점진적으로 보여줄 수 있습니다.

```tsx
import { Suspense } from 'react';

function Page() {
  return (
    <div>
      <h1>블로그</h1>
      <Suspense fallback={<PostsSkeleton />}>
        <Posts />
      </Suspense>
    </div>
  );
}
```

## 실제 프로젝트 구조

제가 선호하는 프로젝트 구조입니다:

```
src/
├── app/                 # 페이지 및 라우팅
│   ├── [locale]/        # i18n 지원
│   └── api/             # API 라우트
├── components/          # 컴포넌트
│   ├── ui/              # 재사용 가능한 UI
│   └── layout/          # 레이아웃 컴포넌트
├── lib/                 # 유틸리티 함수
├── hooks/               # 커스텀 훅
└── types/               # TypeScript 타입
```

## 성능 최적화 팁

### 이미지 최적화

```tsx
import Image from 'next/image';

function OptimizedImage() {
  return (
    <Image
      src="/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
      priority // LCP 이미지에 사용
    />
  );
}
```

### 폰트 최적화

```tsx
import { Pretendard } from 'next/font/local';

const pretendard = Pretendard({
  src: './fonts/Pretendard.woff2',
  display: 'swap',
});
```
